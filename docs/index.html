1 Introduction

Exam Timetable Planner is a standalone desktop application designed to automatically generate optimized exam schedules for educational institutions. The system processes courses, classrooms, student enrollments, and predefined business constraints to produce a conflict-free and efficient exam timetable.

The primary users of this application are Student Affairs personnel responsible for creating and maintaining examination schedules. The application aims to reduce manual scheduling effort and eliminate human errors in the complex process of exam timetabling.

üìå Target Platform

This application is developed as a cross-platform desktop solution compatible with Windows, macOS, and Linux operating systems.

2 Project Objectives

The main objectives of this project are:

  • Automation: Eliminate manual exam scheduling by implementing an intelligent algorithm
  • Constraint Satisfaction: Ensure all scheduling rules and institutional policies are respected
  • User-Friendly Interface: Provide an intuitive GUI for data import, visualization, and manual adjustments
  • Flexibility: Allow manual overrides with real-time constraint validation
  • Scalability: Support large-scale institutions with 10,000+ students
  • Data Export: Enable exporting of finalized timetables for external use

6 Scheduling Algorithm

üí° Algorithm Overview

The core of the application is a Constraint Satisfaction Problem (CSP) solver that uses recursive backtracking. Unlike simple greedy approaches, our solver searches for the globally optimal schedule by minimizing the total number of exam days used.

1. Binary Search for Optimal Days

Instead of just trying to "fit" exams into a fixed range, the algorithm performs a Binary Search on the number of days. For any given range [low, high], it attempts to find a valid schedule for mid days. If successful, it tries fewer days to find the true minimum.

2. Course Splitting (Exam Parts)

To handle large courses that exceed the capacity of any single classroom, the system implements Dynamic Course Splitting:

  • Balanced Partitioning: Students are divided into N parts where N = ceil(Students / MaxRoomCapacity).
  • Simultaneous Scheduling: All parts of a split course are forced into the same time slot across different classrooms to maintain exam integrity.

3. Advanced Heuristics & Pruning

To avoid the exponential time complexity of pure backtracking, we use several heuristics:

Most Constrained First

Largest courses are scheduled first. This reduces the search space early by filling up heavy constraints while resources are still plentiful.

Smallest Fit (Best-Fit)

Classrooms are assigned by selecting the smallest room that fits the group. This preserves larger rooms for courses that actually need them.

Fail-Fast Pruning

The ScheduleState keeps a bit-mapped index of student assignments, allowing O(1) checks for daily limits and time overlaps.

Complexity Analysis

Layer Complexity Technical Detail
Day Search O(log D) Binary search over possible day ranges (max 31 days).
Placement O(N ^ P) Recursive backtracking where P is the number of possible slots.
Validation O(E) E = enrollment count. Using hash-maps for O(1) lookup.
üìå Optimization

The algorithm uses early termination and slot caching to improve performance. Large-scale tests show it handles 10,000+ students efficiently.

Layer Responsibilities

Presentation (UI)

Handles user interactions via JavaFX. Uses FXML for declarative UI and CSS for styling. Responsible for asynchronous task management to keep the UI responsive during heavy scheduling.

Services (Logic)

The "brain" of the app. SchedulerService handles the generate-and-check cycle, while ConstraintChecker enforces academic rules.

Persistence (Data)

Abstracts SQLite operations using the Repository Pattern. Manages connection pooling and ensures transactional integrity during imports.

Domain (Models)

Contains the core business objects. These are immutable plain Java objects that represent the stable data structure of the entire system.

17 Frequently Asked Questions

What file formats are supported for import?

The application supports UTF-8 encoded CSV files for all data imports. Each file type (courses, students, classrooms, enrollments) has a specific format that must be followed. Sample files are provided in the repository.

How does the scheduling algorithm handle conflicts?

The algorithm uses a constraint satisfaction approach with backtracking. It prioritizes courses with higher enrollment and checks all constraints (time window, capacity, student conflicts, daily limits, gap requirements) before assigning each exam. If a conflict is detected, it automatically tries alternative slots.

Can I manually modify the generated timetable?

Yes! After automatic generation, you can drag-and-drop exams to different time slots or classrooms. The system validates all changes in real-time and warns you about any constraint violations.

What is the maximum capacity the system can handle?

The system has been tested with 10,000+ students, 500+ courses, and 100+ classrooms. Performance remains optimal for most educational institutions. For very large datasets, consider splitting the scheduling by department or faculty.

How do I export the final timetable?

Go to the timetable view and click the Export button. You can choose from CSV, Excel, or PDF formats. Options include exporting the master schedule, student-specific schedules, or room-specific schedules.

Is there a way to exclude specific dates from scheduling?

Yes, you can define blackout dates in the scheduling configuration. These dates (such as holidays or special events) will be excluded from the available time slots during timetable generation.
